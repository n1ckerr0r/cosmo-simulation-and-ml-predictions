# 3D визуализация движения тел (Unity)

## Описание проекта

Данный проект представляет собой систему визуализации движения материальных точек в трёхмерном пространстве на основе заранее подготовленных данных. Основной задачей является отображение изменения положения тел во времени и сравнение реальных координат с предсказанными значениями.

Приложение не выполняет физических расчётов, а использует входные данные и отображает их в виде динамической сцены.



## Используемые технологии

* Unity (3D) — среда разработки и визуализации
* C# — основной язык программирования
* CSV (TXT) — формат входных данных
* Стандартные компоненты Unity:

  * GameObject
  * Transform
  * Time.deltaTime
  * динамическое создание примитивов (Sphere)



## Структура проекта

### MotionRecord.cs

Класс, представляющий одну запись данных.

Содержит:

* simId — идентификатор симуляции
* step — временной шаг
* body — идентификатор тела
* truePosition — реальные координаты
* predPosition — предсказанные координаты

Используется для хранения и передачи данных внутри приложения.



### MotionVisualizer.cs

Основной скрипт, отвечающий за работу приложения.

Выполняет следующие функции:

#### Загрузка данных

* Читает CSV-файл из папки StreamingAssets
* Парсит строки файла
* Фильтрует данные по выбранной симуляции
* Формирует список записей MotionRecord

#### Создание объектов

* Для каждого тела создаются два объекта:

  * Body_X_True — отображает реальные координаты (зелёный цвет)
  * Body_X_Pred — отображает предсказанные координаты (красный цвет)
* Объекты создаются автоматически через код

#### Воспроизведение движения

* Движение реализовано через обновление позиций объектов по временным шагам
* Используется таймер (Time.deltaTime)
* Поддерживается управление скоростью воспроизведения



## Формат входных данных

Файл должен находиться в папке:

Assets/StreamingAssets/

Формат CSV:

sim_id,step,body,x_true,y_true,z_true,x_pred,y_pred,z_pred,model_name

Где:

* sim_id — номер симуляции
* step — временной шаг
* body — номер тела
* x_true, y_true, z_true — реальные координаты
* x_pred, y_pred, z_pred — предсказанные координаты



## Принцип работы

1. При запуске сцены:

   * загружается файл с данными
   * выполняется парсинг
   * определяется набор временных шагов

2. Создаются объекты для первого шага

3. Запускается воспроизведение:

   * на каждом шаге обновляются позиции объектов
   * отображается изменение положения тел во времени


## Настраиваемые параметры

Основные параметры доступны в Inspector:

* File Name — имя файла с данными
* Selected Sim Id — выбор симуляции
* Visual Scale — масштаб отображения координат
* Step Duration — интервал между шагами
* Step Skip — ускорение воспроизведения
* True Sphere Size — размер объектов для реальных координат
* Pred Sphere Size — размер объектов для предсказанных координат


## Особенности реализации

* Все объекты создаются автоматически без ручной настройки сцены
* Поддерживается выбор одной из симуляций, содержащихся в файле
* Используется масштабирование координат для корректного отображения больших значений
* Реальные и предсказанные данные визуально различаются по цвету



## Результат

В результате работы реализована 3D-сцена, в которой:

* отображаются положения тел
* реализовано воспроизведение их движения во времени
* обеспечено наглядное сравнение реальных и предсказанных координат

Проект может использоваться для визуального анализа данных и оценки качества предсказаний моделей.
