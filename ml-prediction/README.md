# ML Prediction

## Описание этапа

Цель данного этапа — построить модель машинного обучения, способную предсказывать поведение космических тел на основе данных, полученных в результате численного моделирования.

В качестве входных данных используются CSV-файлы из папки `results/`, полученные на этапе моделирования.

Результатом этапа является:
- обученная модель
- файл с предсказаниями `ml_pred.csv`
- оценка качества модели с использованием метрик

## Используемые данные

Использовались следующие файлы:

- ``program_model1.csv``
- `program_model2.csv`
- `program_model3.csv`

### Структура данных

Каждая строка описывает состояние одного тела на конкретном шаге симуляции:

| Поле   | Описание |
|--------|----------|
| step   | номер шага симуляции |
| body   | идентификатор тела |
| x      | координата X |
| y      | координата Y |
| z      | координата Z |
| energy | энергия системы |

## Особенности данных

В ходе анализа было выявлено:

- данные представляют собой **временные ряды**
- каждая симуляция содержит несколько тел
- координата `z` **всегда равна 0**

## Постановка задачи

Задача сформулирована как задача **регрессии**:

> по текущему состоянию системы на шаге `t` предсказать координаты тела на шаге `t+1`

То есть модель должна предсказывать:

- `x(t+1)`
- `y(t+1)`
- `z(t+1)` (в данном случае константа)

## Подготовка данных

### 1. Объединение данных

Файлы `program_model1.csv`, `program_model2.csv` и `program_model3.csv` объединяются в один датафрейм.

Добавляется признак:

- `sim_id` — идентификатор симуляции (по имени файла)

### 2. Сортировка

Данные сортируются по: `sim_id → body → step`

Это важно для корректной работы с временными рядами.

### 3. Генерация признаков

Для каждого тела создаются дополнительные признаки.

#### Лаги координат

- `x_lag1`, `y_lag1`, `z_lag1` — значения на предыдущем шаге
- `x_lag2`, `y_lag2`, `z_lag2` — значения за два шага до текущего

#### Приближенные скорости
- `vx_approx = x - x_lag1`
- `vy_approx = y - y_lag1`
- `vz_approx = z - z_lag1`

### 4. Формирование целевых переменных

Целевые переменные — координаты на следующем шаге:

- `target_x`
- `target_y`
- `target_z`

Создаются с помощью сдвига (`shift(-1)`).

### 5. Очистка данных

Удаляются строки:
- где отсутствуют лаги
- где отсутствует target

## Разделение на train/test

Разделение выполняется **по времени**:

- первые 80% шагов → обучение
- последние 20% шагов → тест

## Используемые модели

### 1. LinearRegression (baseline)

Используется как простая базовая модель.

Плюсы:
- быстрая
- интерпретируемая

Минусы:
- не умеет моделировать нелинейные зависимости

### 2. CatBoostRegressor

Основная модель.

Причины выбора:
- хорошо работает с табличными данными
- умеет ловить нелинейные зависимости
- не требует сложной подготовки данных

## Обработка константной переменной

Так как: `z = 0` для всех наблюдений

то:

- модель для `target_z` не обучается
- используется константное предсказание `z = 0`

## Метрики качества

Используются:

### MAE (Mean Absolute Error)
Средняя абсолютная ошибка

### RMSE (Root Mean Squared Error)
Корень из средней квадратичной ошибки

### R² (коэффициент детерминации)
Показывает, насколько хорошо модель объясняет данные

## Результаты

### LinearRegression

- MAE  = `1.152358e+11`
- RMSE = `1.783686e+11`
- R²   = `0.293961`

### CatBoostRegressor

- MAE  = `1.096808e+10`
- RMSE = `1.994895e+10`
- R²   = `0.989602`

## Анализ результатов

- линейная модель показала слабый результат (R² ≈ 0.29)
- это связано с нелинейной природой движения тел
- CatBoost значительно улучшил качество
